# XP2測線網格討論
### PyGimli驗證的必要性
+ PyGimli是一個函式庫，所以要如何正確的呼叫函式庫來解決問題是使用者的責任。
+ 針對ERT的部分，PyGimli大量依賴繼承自BERT(Boundless Electrical Resistivity Tomography)的C++函式庫。
  + 這表示，許多的預設值並非由Python端決定，而是定義在C++端。
  + 函式庫可能存在說明不完全的情況，導致誤導使用者。此外，許多預設值的查詢方式並不容易，簡單的呼叫函數常常會有誤解，且不易察覺。
  + 部分功能與期刊論文的描述有落差。也就是程式碼並沒有實作某些功能。
+ 對使用者來說，PyGimli僅是一個工具箱，找出穩定可使用的工具才是務實的做法。因此，將工具拼湊起來後，必須進行驗證。
  
### 建議的驗證方式1: 幾何因子解析解比較 (Analytical Comparison)
+ 驗證的第一步就是用它來解決一個「已知答案」的問題。即：**解析解比較 (Analytical Comparison)：***
  + 建立一個最簡單的「半無限空間均質模型」，先手動以物理公式計算理論電位的解析解 (Analytical)，再執行 pyGIMLi 的順推模擬求得數值解 (Numerical)，觀察解析解 (Analytical)與數值解 (Numerical)的吻合程度。
  + 根據視電阻率定義，「在均質模型下，真實電阻率等於視電阻率」。既然真實電阻率為已知常數，即可透過理論電位反推出電極配置的「幾何因子（K）」。因此，比較均質模型下的理論電位，本質上等同於比較幾何因子的準確性。
  + 最終決定利用「幾何因子」作為關鍵的「已知答案」驗證。這也表示，是在驗證**均質模型條件下**的數值模擬。
  + 是否通過驗證則視使用者需求而定，例如：預期結果差異「小於1%」。根據研究精度要求，也可以設定更嚴格或更寬鬆的容許範圍。
  + 在決定容許範圍之前，應先在計算機可運行的條件下，列舉不同**邊界條件**與**網格密度**所產生的具體差異量。
  + 在決定這個結果差異容許範圍之前，可以先列舉不同邊界條件與不同網格密度在計算機可運行的條件下，具體結果差異量為何。
    + 透過比較不同案例，不僅能了解誤差來源，也能明確知道要付出多大的**運算代價**（記憶體消耗、計算時間）才能達到目標容許範圍。這有助於在精度與效率之間找到最務實的平衡點。

  + 在數值模擬中，選擇數值解(K_mesh)才能與目前使用的網格參數完全匹配，理論上能達到最具自洽性（Self-consistency）的順推模擬結果。

  + 最終建議: 由於幾何因子在網格與座標固定後僅需計算一次，這點運算成本在工程實務上完全可以接受，**優先選擇數值解 (K_mesh) 才是正確的做法**。針對大量資料，建議將計算好的數值解儲存，以便後續反覆調用，兼顧精度與效率。
 
### 建議的驗證方式2: 均質順推模擬驗證 (Homogeneous Forward Modeling Validation)
+ 對於初級使用者來說，他們不好奇選擇的是何種幾何因子。他們在乎的是順推模擬結果是否正確，也就是計算出的視電阻率是否正確。
+ 對於初級使用者而言，幾何因子的選用邏輯通常過於技術化。他們更直觀關心的是：「順推模擬出來的視電阻率是否正確？」
+ 因此，更應該被考慮的驗證方式就是，同樣透過解決一個具有「已知答案」的問題，來驗證逆推工具的「順推運算」部分。
+ 直接比對由模擬產生的視電阻率(Rhoa)與設定的均質背景電阻率。若模擬出的視電阻率(Rhoa)穩定落在均質背景值附近，則證明該「順推運算」與當下網格設定足以支撐後續的反演計算。
+ 事實上，當驗證方法2(均質順推模擬驗證)通過，等同於選擇好幾何因子的驗證方法1(幾何因子解析解比較)也已經通過。
  + 因果關係：視電阻率是透過電位（模擬結果）與幾何因子（網格特徵）共同計算得出的。如果視電阻率(Rhoa)與設定的均質背景電阻率偏差極小，代表幾何因子與順推電場模擬都已在該網格環境下達成正確的對應。
  + 實務意義：
    + 對於一般使用者，僅需展示驗證方式2(均質順推模擬驗證)通過即可建立信任。
    + 對於開發者，則需透過驗證方式1(幾何因子解析解比較)確保底層邏輯的自洽性（Self-consistency）。

### 建議的驗證方式#2
+ 合成模型測試 (Synthetic Model Testing)：設計一個簡單的異常體（如方塊或球體），加入預期的隨機雜訊，測試您的反演流程是否能還原目標。

### 無地形的時候幾何因子影響
+ 選解析解(K0)時，順推算出的是電阻率幾乎正確，與正確值差異小於+-0.01%。9種網格都是。
+ 選數值計算(K_mesh)時，順推算出的是電阻率幾乎正確，與正確值差異小於+-0.01%。9種網格都是。
+ K0與K_mesh幾乎相同。
  + 表示數值模擬正確性高。
  + 地層不複雜的時候，使用粗的網格可以有效節省運算時間，同樣也節省記憶體。

### 有地形的時候(XP2)幾何因子影響
+ 選解析解(K0)時，順推算出的視電阻率差異很大，差異最大可達+-50%，等於電阻率錯估為正確值的50%~150%。9種網格都是。
  + 當順推偏差過大（例如: 50% ~ 150%）時，將直接導致逆推問題進入不可控制的狀態：
    + 收斂困難 (Convergence Failure)：
      + 殘差無法消除：逆推演算的核心是最小化觀測值與理論值的差異。由於 K0 本身帶來的系統誤差，觀測值與模型理論值之間存在巨大的「天然鴻溝」，導致目標函數永遠無法達到收斂標準。
      + 陷入局部極小值：演算法可能在錯誤的方向上反覆震盪，始終無法找到真實的模型參數。
    + 系統偏差與偽影 (Systematic Artifacts)：
      + 數值補償現象：為了強行填補 K0 造成的 50% 或 150% 誤差，逆推程式會錯誤地調整模型中的電阻率分布。
      + 偽影產生：在勉強收斂的情況下，結果中會出現不存在的高阻或低阻異常體。這些「偽影」純粹是為了抵消幾何因子的誤差而產生的數學湊數，而非地質結構。
    + 結果不可解釋 (Uninterpretable Results)：
      + 特徵扭曲：由於順推模型已經嚴重偏離物理真實情況，預期的地質特徵（如地層界面、斷層位置）會被拉伸、壓縮或移位。
      + 錯誤決策：解釋人員若基於被扭曲的剖面進行分析，將導致嚴重的誤導。
+ 選數值計算(K_mesh)時，順推算出的是電阻率幾乎正確，與正確值差異小於+-0.01%。9種網格都是。
+ 地形效應(Topography Effect):
  + Pygimli作者定義了地形效應(Topography Effect): 
    + Topography Effect = k_FlatEarth/ k_Simulated
    + k_FlatEarth: 理論上的平坦大地幾何因子
    + k_Simulated: 數值模擬算出的真實幾何因子。
  + 這種效應非常強烈，在作者的3D案例中表示，導致視電阻率產生 3 到 4 倍的增加或減少。
  + 作者建議:
    + 如果比值遠離 1，說明地形效應非常嚴重，必須在反演前進行修正，否則會在地底解釋出假的人造構造。
  + REF:
    + Günther, T., Rücker, C., & Spitzer, K. (2006). Three-dimensional modelling and inversion of DC resistivity data incorporating topography—II. Inversion. Geophysical Journal International, 166(2), 506-517.
